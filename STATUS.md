## Phases

### ✅ Phase 1 — Audits and Coverage Reporting
**Status: Done**

- Built-in decklists resolve through the Scryfall importer and into `auditImportedDeck`
- Markdown report generated by `npm run audit:decks`
- Cards tagged by support level (automated / partial / manual / unsupported), candidate type (generic / bespoke), and confidence (parser-match / runtime-verified / manual-only)
- Top gaps ranked by frequency × deck-importance tier
- Bespoke card registry (`src/lib/card-support/index.ts`) formalised

---

### ✅ Phase 2A — Shared Target Picker and Pending Choice Flow
**Status: Done**

- Generic `pendingTargetChoice` model replaces ad-hoc pickers
- Legal-target filtering through shared helpers (battlefield creature, nonland permanent, planeswalker, graveyard creature, stack spell, creature-or-player)
- Chooser can dismiss and reopen the picker from a persistent banner
- Targeted triggers and targeted spells use the same pending-target model
- Pending choices survive multiplayer state sync
- Abilities (planeswalker, explore) go through the shared target flow

---

### ✅ Phase 2B — Manual Stack Resolution
**Status: Done**

- Any unsupported spell can be cast and manually resolved without blocking the game
- Stack items expose "Resolve manually" for unsupported / partial spells
- "Remove from stack / counter / cancel" controls available for admin / test mode
- Manual resolution is logged and undoable
- Counterspell stack targeting supported

---

### ✅ Phase 2C — Generic Spell Pattern Expansion
**Status: Done**

Exit criteria:

| Criterion | Target | Actual |
| --- | --- | --- |
| Unsupported instants/sorceries — Hazel Squirrels | ≤ 2 | **2** ✓ |
| Unsupported instants/sorceries — Auntie Ool Blight | ≤ 2 | **2** ✓ |
| Pool instants/sorceries runtime-verified automated | ≥ 80% | **~84%** ✓ |
| Unsupported spells castable via "resolve manually" | All | ✓ (Phase 2B) |

Patterns added in this phase:
- `copy_all_tokens` — Second Harvest ("for each token you control, create a copy")
- `remove_all_counters_draw_lose_life` — Eventide's Shadow (simplified: removes all counters from all permanents, draw/lose equal)
- "put creature cards from graveyards onto battlefield" — Aberrant Return (simplified to single target, with -1/-1 counter flag)
- Oracle-text snapshot tests for all automated instants/sorceries from both test decks
- Blanket `simple-spell` runtime-verified verification covers ~84% of pool instants/sorceries

**Remaining unsupported instants/sorceries (by design):**

| Card | Deck | Why deferred |
| --- | --- | --- |
| Promise of Aclazotz // Foul Rebirth | Hazel | MDFC adventure; requires token populate + sacrifice-as-cost |
| Rootcast Apprenticeship | Hazel | Multi-modal with repeated modes; no multi-target support yet |
| Burning Curiosity | Blight | Exile-top-N + play-until-end-of-turn; too complex |
| Fire Covenant | Blight | Pay-X-life + divide damage among N targets; needs multi-targeting |

### 🔲 Phase 3 — Generic Activated Ability System
**Status: Not started**

**Prerequisite:** Extract cost/effect dispatch from `game-reducer.ts` (~5,400 lines) into its own module before adding new abilities. This is the first task in Phase 3.

Goals:
- Cost primitives: tap source, pay mana, pay life, sacrifice source / another creature / an artifact, discard, remove counter, tap N untapped creatures
- Effect primitives: add mana, draw, gain life, create tokens, put counters, deal damage, destroy target, search basic land, explore, scry, surveil
- Mana rocks, dorks, token artifacts, simple sac outlets, and tap abilities all resolve through shared machinery
- Adding a typical activated ability requires no reducer changes

Exit criteria:
- ≥ 80% of activated abilities across the pool are runtime-verified
- No Tier 1 deck has an unsupported activated ability on a centerpiece (commander, mana base, primary engine)

---

**Status: Not started**

- No Tier 1 deck has an unsupported trigger on a centerpiece

### 🔲 Phase 5 — Static Effects and Combat Keywords

### 🔲 Phase 6 — Commander-Specific Completion
**Status: Partial (existing)**

Existing: commander tax, cast from command zone, commander damage tracking.  
Remaining:
- Zone-change replacement choice (death → command zone vs graveyard)
- Bespoke support for Hazel of the Rootbloom (commander, currently partial)
- Bespoke support for Auntie Ool, Cursewretch (commander, currently manual/blocker)

Exit criteria:
- A commander-centric deck plays a full game without commander rules being mostly manual

---

### 🔲 Phase 7 — Multiplayer Trust
**Status: Not started**

Goals: reconnect reliability, host/admin authority, hidden information (hand/library privacy), M2 manual-correction authority model.

---

### 🔲 Phase 8 — Larger Rules Subsystems
**Status: Deferred**

Counterspells, replacement effects, prevention, copy spells, APNAP, layers, MDFCs, alternate costs, flash restrictions, full planeswalker support.
